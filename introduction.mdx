# QIMEM — Deterministic Encryption & Key Lifecycle Engine

QIMEM is a focused encryption engine for deterministic, versioned, auditable envelope encryption and key lifecycle management.

## What QIMEM Is

A production-ready cryptographic engine that provides:

- **Deterministic envelope encryption** with versioned format
- **Key lifecycle management** with rotation support
- **Auditable operations** with structured error handling
- **Multiple deployment modes** (in-memory, PostgreSQL-backed)
- **HTTP API and CLI** for flexible integration

## What QIMEM Is Not

QIMEM intentionally does not provide:

- Authentication or identity management
- Authorization or access control
- Plugin frameworks or extensibility layers

This focused scope ensures reliability and security through simplicity.

## Architecture Components

### Core Library (`qimem`)

The foundation providing:
- Crypto engine and envelope format
- Key store traits and implementations
- In-memory key store (default)
- PostgreSQL key store (`stateful` feature)

### HTTP API (`qimem-api`)

Axum-based REST API exposing:
- Key lifecycle operations (create, rotate)
- Encryption and decryption endpoints
- Health checks and error handling

### CLI (`qimem`)

Command-line interface for:
- Key generation
- File and data encryption/decryption
- Key rotation workflows

## Envelope Format

QIMEM uses a deterministic envelope format (v1) with these fields:

- `version: u8` — Envelope format version (currently 1)
- `algorithm: Algorithm` — Encryption algorithm identifier
- `key_id: Uuid` — Reference to encryption key
- `nonce: Vec<u8>` — Encryption nonce
- `ciphertext: Vec<u8>` — Encrypted payload
- `tag: Vec<u8>` — Authentication tag

<Info>
The envelope supports both **binary** and **JSON** serialization formats, both deterministic for consistency.
</Info>
