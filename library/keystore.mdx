## Overview

The `KeyStore` trait defines the interface for managing encryption keys. QIMEM provides two built-in implementations: `InMemoryKeyStore` for testing and stateless deployments, and `PostgresKeyStore` for production stateful storage.

## KeyStore Trait

The `KeyStore` trait provides methods for key lifecycle management.

```rust
pub trait KeyStore: Send + Sync {
    /// Create a new root key
    fn create_key(&self) -> Result<KeyMetadata>;
    
    /// Retrieve key material by key id
    fn get_key(&self, key_id: Uuid) -> Result<KeyMaterial>;
    
    /// Rotate a key and return the new active version
    fn rotate_key(&self, key_id: Uuid) -> Result<KeyMetadata>;
}
```

### Methods

#### create_key

Creates a new encryption key with a unique key ID and lineage ID.

```rust
fn create_key(&self) -> Result<KeyMetadata>
```

**Returns:** `Result<KeyMetadata>` containing metadata for the newly created key.

**Example:**

```rust
use qimem::{InMemoryKeyStore, KeyStore};

let store = InMemoryKeyStore::default();
let key_meta = store.create_key().expect("create key");

println!("Key ID: {}", key_meta.key_id);
println!("Lineage ID: {}", key_meta.lineage_id);
println!("Version: {}", key_meta.version);
println!("Active: {}", key_meta.active);
```

#### get_key

Retrieves key material by key ID.

```rust
fn get_key(&self, key_id: Uuid) -> Result<KeyMaterial>
```

<ParamField path="key_id" type="Uuid" required>
  The UUID of the key to retrieve.
</ParamField>

**Returns:** `Result<KeyMaterial>` containing the key bytes and metadata.

**Errors:**
- `QimemError::KeyNotFound` if the key ID does not exist

**Example:**

```rust
use qimem::{InMemoryKeyStore, KeyStore};

let store = InMemoryKeyStore::default();
let key_meta = store.create_key().expect("create key");

let key_material = store.get_key(key_meta.key_id).expect("get key");
println!("Key material length: {} bytes", key_material.material.len());
```

#### rotate_key

Rotates a key by marking the current version inactive and creating a new active version in the same lineage.

```rust
fn rotate_key(&self, key_id: Uuid) -> Result<KeyMetadata>
```

<ParamField path="key_id" type="Uuid" required>
  The UUID of the key to rotate.
</ParamField>

**Returns:** `Result<KeyMetadata>` containing metadata for the new active key version.

**Errors:**
- `QimemError::KeyNotFound` if the key ID does not exist

**Example:**

```rust
use qimem::{Algorithm, CryptoEngine, InMemoryKeyStore, KeyStore};

let store = InMemoryKeyStore::default();
let engine = CryptoEngine::new(Algorithm::Aes256Gcm);

// Create initial key and encrypt data
let old_key_meta = store.create_key().expect("create key");
let old_key = store.get_key(old_key_meta.key_id).expect("get key");
let envelope = engine.encrypt(&old_key, b"secret").expect("encrypt");

// Rotate the key
let new_key_meta = store.rotate_key(old_key_meta.key_id).expect("rotate");

// Old key is now inactive
let old_key_after = store.get_key(old_key_meta.key_id).expect("get old key");
assert!(!old_key_after.active);

// Old key can still decrypt old envelopes
let plaintext = engine.decrypt(&old_key_after, &envelope).expect("decrypt");
assert_eq!(plaintext, b"secret");

// But cannot encrypt new data
let err = engine.encrypt(&old_key_after, b"new data").expect_err("should fail");
```

## Data Types

### KeyMetadata

Metadata describing a key without exposing the key material.

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct KeyMetadata {
    /// Key identifier
    pub key_id: Uuid,
    /// Lineage identifier (shared across rotations)
    pub lineage_id: Uuid,
    /// Rotation version number
    pub version: i32,
    /// Whether key is active for encryption
    pub active: bool,
}
```

<ParamField path="key_id" type="Uuid">
  Unique identifier for this specific key version.
</ParamField>

<ParamField path="lineage_id" type="Uuid">
  Shared identifier for all versions of a rotated key.
</ParamField>

<ParamField path="version" type="i32">
  Incrementing version number starting at 1.
</ParamField>

<ParamField path="active" type="bool">
  Whether this key can be used for new encryption operations.
</ParamField>

### KeyMaterial

Actual key bytes with metadata, used for encryption and decryption.

```rust
#[derive(Debug, Clone)]
pub struct KeyMaterial {
    /// Key identifier
    pub key_id: Uuid,
    /// Encryption key bytes (zeroized on drop)
    pub material: Zeroizing<Vec<u8>>,
    /// Active status
    pub active: bool,
}
```

<ParamField path="key_id" type="Uuid">
  The key identifier matching the envelope's key_id.
</ParamField>

<ParamField path="material" type="Zeroizing<Vec<u8>>">
  The raw key bytes (32 bytes for AES-256 and ChaCha20). Automatically zeroized when dropped for security.
</ParamField>

<ParamField path="active" type="bool">
  Whether this key can be used for encryption. Inactive keys can still decrypt.
</ParamField>

## InMemoryKeyStore

A thread-safe in-memory key store suitable for testing and stateless applications.

```rust
pub struct InMemoryKeyStore { /* private fields */ }
```

### Usage

```rust
use std::sync::Arc;
use qimem::{InMemoryKeyStore, KeyStore};

let store: Arc<dyn KeyStore> = Arc::new(InMemoryKeyStore::default());

let key = store.create_key().expect("create key");
let material = store.get_key(key.key_id).expect("get key");
```

### Implementation Notes

- Keys are stored in memory using `Arc<RwLock<HashMap>>`
- Thread-safe and supports concurrent operations
- All keys are lost when the process terminates
- Suitable for testing, ephemeral workloads, and stateless deployments

## PostgresKeyStore

<Note>
Requires the `stateful` feature flag: `cargo add qimem --features stateful`
</Note>

A production-ready key store backed by PostgreSQL.

```rust
#[cfg(feature = "stateful")]
pub struct PostgresKeyStore { /* private fields */ }
```

### Usage

```rust
use std::sync::Arc;
use qimem::{KeyStore, PostgresKeyStore};
use sqlx::PgPool;

#[tokio::main]
async fn main() {
    let pool = PgPool::connect("postgres://user:pass@localhost/qimem")
        .await
        .expect("connect to postgres");
    
    let store: Arc<dyn KeyStore> = Arc::new(
        PostgresKeyStore::new(pool).await.expect("create store")
    );
    
    let key = store.create_key().expect("create key");
    let material = store.get_key(key.key_id).expect("get key");
}
```

### Schema

The `PostgresKeyStore` requires a `keys` table:

```sql
CREATE TABLE keys (
    key_id UUID PRIMARY KEY,
    lineage_id UUID NOT NULL,
    version INTEGER NOT NULL,
    material BYTEA NOT NULL,
    active BOOLEAN NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_keys_lineage ON keys(lineage_id);
```

### Implementation Notes

- Uses SQLx for async PostgreSQL access
- Key material is stored encrypted at rest (database-level encryption recommended)
- Supports concurrent access from multiple application instances
- Key rotation is transactional (marks old key inactive and creates new key atomically)

## Key Rotation Strategy

QIMEM supports key rotation without re-encrypting existing data:

1. **Create initial key**: `create_key()` generates a new key with version 1
2. **Encrypt data**: Use the active key to create envelopes
3. **Rotate**: `rotate_key()` creates version 2 with the same lineage_id
4. **Old data**: Still decryptable with the version 1 key (now inactive)
5. **New data**: Encrypted with version 2 (the new active key)

```rust
use qimem::{Algorithm, CryptoEngine, InMemoryKeyStore, KeyStore};

let store = InMemoryKeyStore::default();
let engine = CryptoEngine::new(Algorithm::Aes256Gcm);

// Initial key
let v1_meta = store.create_key().expect("create");
let v1_key = store.get_key(v1_meta.key_id).expect("get");
let old_envelope = engine.encrypt(&v1_key, b"old data").expect("encrypt");

// Rotate
let v2_meta = store.rotate_key(v1_meta.key_id).expect("rotate");
let v2_key = store.get_key(v2_meta.key_id).expect("get v2");

// New encryptions use v2
let new_envelope = engine.encrypt(&v2_key, b"new data").expect("encrypt");

// Old envelopes still work with v1 (now inactive)
let v1_inactive = store.get_key(v1_meta.key_id).expect("get v1");
let old_plaintext = engine.decrypt(&v1_inactive, &old_envelope).expect("decrypt");
assert_eq!(old_plaintext, b"old data");

// v1 cannot encrypt
let err = engine.encrypt(&v1_inactive, b"fail").expect_err("inactive key");
```

## Thread Safety

Both `InMemoryKeyStore` and `PostgresKeyStore` implement `Send + Sync`, making them safe to share across threads using `Arc<dyn KeyStore>`:

```rust
use std::sync::Arc;
use qimem::{InMemoryKeyStore, KeyStore};

let store: Arc<dyn KeyStore> = Arc::new(InMemoryKeyStore::default());

let store_clone = Arc::clone(&store);
let handle = std::thread::spawn(move || {
    store_clone.create_key().expect("create in thread")
});

let key = handle.join().expect("join");
println!("Created key from thread: {}", key.key_id);
```
